Analysis of Disease Outbreaks

This website uses data from Project Tycho and several data mining algorithms to provide you with results that may be helpful in preventing outbreaks of disease. Click on one of the data mining tasks in the navigation bar above to start exploring the data.
The different data mining tasks are described below:


Outlier Detection

- Outlier detection in our application aims to find cities with unusual statistics for fatality count, occurence count, or fatality percentage for a given disease
- First select a disease that you want to investigate, or press the button to find information for all diseases. Note that this option does take a few minutes to run.
- The KNN algorithm will run for k=2, k=3, and k=4 to find outlier cities for the given disease. The city values will also be compared to the normal distribution and marked as outliers if their z-score is outside the range of 1.5.
- 6 tables will be shown as a result. Each table contains the outlier cities and their relevant information. There will be one table for each of the two algorithms and for each of the three values: occurrence count, fatality count, and fatality percentage. This can be used to infer which cities may be unusual with respect to the selected disease.

Association Rule Mining

- This data mining task finds rules in the data of the form X -> Y where X and Y are different sets of diseases. These rules say that areas that experience outbreaks of diseases in X tend to also experience outbreaks of diseases in Y in a similar time frame. For example, US states that have outbreaks of influenza and typhoid fever in a year also tend to have an outbreak of scarlet fever that year.
- This task takes two parameters which are restrictions on the rules that can be included in the results. The first is minimum support. This is the lower bound on the ratio of data points (time, location combinations) that experienced outbreaks of diseases in all diseases in X and Y to the total number of data points. This limits the returned rules to ones that occur with some frequency. The second parameter is minimum confidence and this limits results to rules that have some level of accuracy. This is the lower bound on the proportion of data points with outbreaks of all the diseases in X that also have outbreaks of all the diseases in Y.

Similarity Mining

- This data mining task determines the similarity of two diseases based on location, date, and the number of fatal and nonfatal cases. Each factor is considered equally.
- The user chooses two diseases to compare, a date range, and a location, and the algorithm finds the similarity between those diseases based on records for that location which fall within that date range.

Clustering

- This data mining task clusters states by disease frequency count using K-means. Additionally, it displays the displays the results on a choropleth map visualization. With this we hope to provide the ability to identify new patterns through visual cluster analysis. User can modify parameters K (number of clusters to identify) and select which decade to cluster by.
